# T0.1: 材质数据模型与种子数据

## Summary

建立材质枚举表与商品数据模型，生成种子数据，为全站按材质分类、筛选、展示提供数据基础。

## PM

- **里程碑**: M1
- **依赖**: 无（首个任务）
- **风险 / 备注**: 材质枚举需与 UX 的筛选标签保持一致；后续新增材质应低成本。

## BA — 用户故事

**US-0.1** 作为 **系统管理者**，我需要一套材质枚举与商品数据模型，以便前台可以按材质对胸针进行分类展示。

### 验收标准

- [x] 材质表包含至少 4 种：珍珠、贝母、纯银、锆石
- [x] 商品与材质之间存在关联（单选或多选）
- [x] 可通过种子数据生成至少 8 件示例商品（实际 10 件）

## UX

- **相关页面 / 流程**: 首页材质入口、列表页筛选、商详页材质标签
- **线框 / 布局**: 参见 design-specs.md — MaterialTag 组件
- **令牌 / 组件**: MaterialTag

## TL — 技术

- **ADR 引用**: —
- **实现备注**: 创建 `Material` 表（id, name, slug, icon?）；`Product` 表含 material 外键或多对多关系；种子脚本填充 ≥8 商品
- **不纳入**: 材质图标管理后台

## QA

- **TCs**: TC-0.1
- **边界用例**: 商品无材质关联时如何处理；材质名为空
- **质量门禁**: 种子数据加载后可通过 API 或 DB 查询验证

## Definition of done

- [x] 验收标准全部满足
- [x] TC-0.1 执行通过
- [x] dashboard / tech-tasks 状态同步更新
