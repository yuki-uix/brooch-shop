# T4.3: 客户端 addToCart 工具接入

## Summary

在 Chat Panel 的 `useChat` 配置中注册客户端工具 `addToCart`，通过 `onToolCall` 回调直接调用 Zustand store action，使用户说「帮我把第一个加入购物车」后助手完成操作并反馈确认，购物车 badge 同步更新。

## PM

- **里程碑**: M5
- **依赖**: T4.2（服务端工具与 API route 就绪，保证 LLM 能识别商品 ID）, T1.2（Zustand cart store）
- **风险 / 备注**: R4（addToCart 误放服务端）——严格遵循 ADR-5，此工具无 `execute`，只在 `onToolCall` 客户端回调中执行。

## BA — 用户故事

**US-4.4** 作为 **用户**，我说「帮我把第一个加入购物车」，助手完成操作并给出确认，购物车 badge 同步变化。

### 验收标准

- [ ] 助手调用 `addToCart` 客户端工具，Zustand store 立即更新

## TL — 技术

- **ADR 引用**: ADR-5（客户端工具执行位置）
- **实现备注**:
  - `ShoppingChatPanel.tsx` 中的 `useChat` 配置:
    ```ts
    const { messages, ... } = useChat({
      api: '/api/chat/shopping',
      onToolCall: async ({ toolCall }) => {
        if (toolCall.toolName === 'addToCart') {
          const { productId, quantity } = toolCall.args;
          // 验证 productId 有效性
          const success = useCartStore.getState().addItem({ productId, quantity });
          return success ? { success: true } : { success: false, error: 'Invalid product' };
        }
      },
    });
    ```
  - `lib/ai/shopping-tools.ts` 中 `addToCart` 工具定义：只有 `description` 和 `parameters`（Zod schema），**无 `execute`**
  - `lib/store/cart.ts`: 确认 `addItem` action 支持通过 productId 添加（已有 T1.2，按需小幅扩展）
- **不纳入**: 加购后跳转到购物车页（用户仍在 Chat 中）；定制行通过 Agent 加购（MVP 范围外）

## QA

- **TCs**: TC-4.3
- **边界用例**:
  - 传入不存在的 productId，助手应返回错误原因
  - 连续两次加购同一商品，数量应累加而非覆盖
  - 加购成功后 badge 数字是否与 store 数量同步
  - 在服务端工具调用完成前用户触发 addToCart，不应崩溃
- **质量门禁**: `addToCart` 回调中无任何 Prisma/DB 调用；badge 动画在 250ms 内触发
