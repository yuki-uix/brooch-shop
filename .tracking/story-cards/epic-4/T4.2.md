# T4.2: 服务端工具 + /api/chat/shopping route

## Summary

实现购物助手的 AI 后端入口：创建 `/api/chat/shopping/route.ts`，注册 `searchProducts` 与 `getProductDetails` 两个服务端工具，使 LLM 可查询 DB 并将结构化结果流式返回给前端。

## PM

- **里程碑**: M5
- **依赖**: T0.1（商品/材质数据模型），T4.1（Chat Panel，提供 API 端点消费方）
- **风险 / 备注**: R2（LLM API 不稳定），需提供 Mock provider 可随时切换；R5（幻觉导致错误商品信息），卡片数据只渲染工具返回值。

## BA — 用户故事

**US-4.2** 作为 **用户**，我可以用自然语言描述需求，助手返回匹配的商品列表。

### 验收标准

- [x] 输入「适合婚礼 + 珍珠 + 300以内」，助手调用 `searchProducts` 工具并以卡片行展示结果
- [x] 每张卡片含：图片、名称、材质标签、价格
- [x] 搜索结果为空时，助手给出友好回应并提供替代建议
- [x] 工具调用过程中显示「正在搜索商品…」中间状态

## TL — 技术

- **ADR 引用**: ADR-5（服务端工具执行位置）, ADR-4（多模型可切换）, ADR-6（useChat in-memory）
- **实现备注**:
  - `app/api/chat/shopping/route.ts`:
    - 使用 `streamText`（Vercel AI SDK），`maxSteps: 5`
    - System prompt 注入：助手角色设定为胸针购物顾问
    - 接收 `messages` 数组（历史对话）实现多轮上下文
  - `lib/ai/shopping-tools.ts`:
    - `searchProducts` — 参数: `{ query: string, material?: string, maxPrice?: number }`；在 `execute` 中调用 Prisma 查询，返回商品数组
    - `getProductDetails` — 参数: `{ productId: string }`；返回单商品完整信息
    - 两个工具均有 `execute`（服务端执行，DB 不暴露客户端）
  - Mock 模式: 通过 `AI_PROVIDER=mock` 环境变量切换固定返回
- **不纳入**: `addToCart` 工具（客户端工具，归 T4.3）；商品卡片 UI 渲染（归 T4.4）

## QA

- **TCs**: TC-4.2
- **边界用例**:
  - 搜索词无匹配商品（空结果）
  - `getProductDetails` 传入不存在的 productId
  - LLM API 超时 / 返回错误时的降级处理
  - messages 数组含 20+ 轮历史时上下文是否正确传递
- **质量门禁**: `searchProducts` 工具返回数据结构与 `ChatProductCard` 所需字段对齐
