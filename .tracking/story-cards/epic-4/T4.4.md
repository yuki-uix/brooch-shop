# T4.4: Chat 内商品卡片渲染

## Summary

实现 `ChatProductCard` 组件，将 `searchProducts` / `getProductDetails` 工具返回的结构化商品数据渲染为 Chat 消息流中的内联卡片（图片 + 名称 + 材质标签 + 价格），替代纯文字展示，让用户直观浏览搜索结果。

## PM

- **里程碑**: M5
- **依赖**: T4.2（工具返回数据结构已确定）
- **风险 / 备注**: R5（LLM 幻觉）——卡片所有字段必须来自工具返回值，不得从 LLM 文字中解析数字或价格。

## BA — 用户故事

**US-4.2**（延伸）作为 **用户**，我可以用自然语言描述需求，助手以商品卡片形式返回匹配结果。

### 验收标准

- [ ] 输入「适合婚礼 + 珍珠 + 300以内」，助手调用 `searchProducts` 工具并以卡片行展示结果
- [ ] 每张卡片含：图片、名称、材质标签、价格
- [ ] 搜索结果为空时，助手给出友好回应并提供替代建议

## TL — 技术

- **ADR 引用**: ADR-5（卡片数据只渲染工具返回值，不解析 LLM 文本）
- **实现备注**:
  - `app/components/ChatProductCard.tsx` — 接受 `product: ProductSummary` prop，渲染图片/名称/材质/价格；内置「加入购物车」按钮（直接调用 `useCartStore.addItem`，客户端操作）
  - `app/components/ChatMessage.tsx` 扩展：检测 message 中的 `toolInvocations`，若 `toolName === 'searchProducts'`，渲染卡片列表；`toolName === 'getProductDetails'`，渲染单卡片大图
  - `ProductSummary` 接口: `{ id, name, material, price, imageUrl }`（与 T4.2 工具返回对齐）
  - 图片降级处理：`imageUrl` 为空时显示占位图
- **不纳入**: 卡片内直接触发询价/定制（MVP 范围外）；卡片点击跳转商详（可选，不阻塞本 task）

## QA

- **TCs**: TC-4.4
- **边界用例**:
  - `searchProducts` 返回空数组时，不渲染卡片，由 LLM 文字兜底
  - 图片 URL 失效时占位图正常展示
  - 卡片内「加入购物车」点击后 badge 同步更新（与 T4.3 联调）
  - 卡片列表超过 5 条时，布局不溢出 Panel 宽度
- **质量门禁**: 卡片价格字段 100% 来自工具返回值（reviewer 验证无 LLM 文本解析）
