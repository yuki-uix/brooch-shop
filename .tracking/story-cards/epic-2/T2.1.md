# T2.1: 定制入口与输入界面

## Summary

实现定制页面入口及输入界面，支持图片上传和/或文字描述提交，为 AI 可行性判断提供输入端。

## PM

- **里程碑**: M3
- **依赖**: T0.1（基础导航与页面结构）
- **风险 / 备注**: 图片上传需限制大小与格式，防止 AI API 调用成本失控。

## BA — 用户故事

**US-2.1** 作为 **用户**，我希望通过「定制」入口上传图片或输入文字描述，以便发起定制胸针请求。

### 验收标准

- [x] 首页/导航有明确的「定制」入口
- [x] 定制页支持上传图片（JPG/PNG，≤5MB）
- [x] 定制页支持文字描述输入（≤500字）
- [x] 图片和文字可单独使用或组合提交
- [x] 空提交时给出提示

## UX

- **相关页面 / 流程**: 定制页 — 输入态
- **线框 / 布局**: 参见 design-specs.md — 定制页输入态线框

```
+------------------------------------------+
|  📷 上传参考图片                          |
|  +----------------------------------+   |
|  |  拖拽或点击上传 (JPG/PNG ≤5MB)    |   |
|  +----------------------------------+   |
|                                          |
|  ✏️ 描述你想要的胸针                      |
|  +----------------------------------+   |
|  |  例如：一枚银色底座搭配淡水珍珠    |   |
|  |  的蝴蝶造型胸针...               |   |
|  +----------------------------------+   |
|                                          |
|  [       提交定制需求       ]             |
+------------------------------------------+
```

- **令牌 / 组件**: FileUploader, TextArea（带字数统计）

## TL — 技术

- **ADR 引用**: ADR-1
- **实现备注**: 前端文件上传组件（drag & drop + click）；图片客户端压缩或直传（视技术栈）；提交 API `POST /api/custom-requests`
- **不纳入**: 多图上传、草稿保存

## QA

- **TCs**: TC-2.1
- **边界用例**: 非图片文件上传；超过 5MB；超过 500 字；快速重复提交
- **质量门禁**: 三种输入组合（仅图/仅文/图文）均可成功提交

- [x] 验收标准全部满足
- [x] TC-2.1 执行通过（与实现一致的预期：见 test-plan.md）
