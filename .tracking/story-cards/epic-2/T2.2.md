# T2.2: AI 可行性判断与报价服务

## Summary

实现后端 AI 服务，接收图片/文字输入，返回结构化的可行性判断（可制作/需确认/不建议）、理由和报价金额；前端展示结果与免责声明。

## PM

- **里程碑**: M3
- **依赖**: T2.1（输入界面提供提交数据）
- **风险 / 备注**: R2（AI API 不稳定）需通过 Mock 实现缓解；报价不具法律约束力需前台声明。

## BA — 用户故事

**US-2.2** 作为 **用户**，我提交定制需求后，希望看到 AI 给出的可行性结论和报价，以便决定是否继续。

### 验收标准

- [x] 提交后展示 loading 状态
- [x] 返回可行性判断：「可制作」/「需确认」/「不建议制作」三级
- [x] 返回简短理由说明
- [x] 返回报价金额（数字）或价格区间
- [x] 页面包含免责声明
- [x] API 超时/失败时展示友好错误提示

## UX

- **相关页面 / 流程**: 定制页 — 结果态
- **线框 / 布局**: 参见 design-specs.md — 定制页结果态线框
- **令牌 / 组件**: FeasibilityBadge（三色）, PriceSnapshot, 免责声明文本

## TL — 技术

- **ADR 引用**: ADR-1（可解释性优先）, ADR-4（多模型可切换）
- **实现备注**:
  - AI 服务接口: `POST /api/custom-requests/:id/evaluate`
  - 响应结构（Zod schema 强制）: `{ feasibility: 'feasible' | 'needs_confirmation' | 'not_recommended', reason: string, quoteAmount: number }`
  - 统一调用层: Vercel AI SDK `generateObject()` + Zod schema
  - 可用 provider / 模型:
    - `openai` — gpt-4o / gpt-4o-mini
    - `google` — gemini-2.5-pro / gemini-2.0-flash
    - `anthropic` — claude-sonnet-4-20250514 / claude-3-5-sonnet-20241022
    - `mock` — 本地固定返回（离线开发/测试）
  - 切换方式: 环境变量 `AI_PROVIDER` + `AI_MODEL`（无需改代码）
  - 关键文件: `lib/ai/providers.ts`（注册）, `lib/ai/evaluator.ts`（调用）
  - `CustomRequest` 模型存储: imageUrl, description, feasibility, reason, quoteAmount, provider, model, createdAt
- **不纳入**: 多轮对话；人工审核覆盖；报价与工时/BOM 挂钩；前台选择模型（MVP 仅后台配置）

## QA

- **TCs**: TC-2.2
- **边界用例**: AI 返回非法 JSON；超时 >10s；可行性为 needs_confirmation 或 not_recommended 时不应显示加购按钮
- **质量门禁**: 三种可行性结果各至少测试 1 次；Mock ↔ 真实 provider 切换仅改 `.env`，无代码改动；至少用两个不同 provider 验证输出结构一致
