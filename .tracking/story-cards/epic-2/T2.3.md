# T2.3: 定制项接入购物车与结账

## Summary

将「可制作」的定制项作为特殊行项目加入购物车，复用标准品 checkout 流程，下单时锁定报价快照。

## PM

- **里程碑**: M3
- **依赖**: T2.2（AI 返回报价）, T1.2（购物车）, T1.3（结账）
- **风险 / 备注**: R3（定制行与标准品行模型不一致）需严格遵循 ADR-2。

## BA — 用户故事

**US-2.3** 作为 **用户**，我接受 AI 报价后，希望将定制项加入购物车或直接结账，以便完成定制下单。

### 验收标准

- [x] 「可制作」结果旁有「加入购物车」/「立即结账」按钮
- [x] 定制行在购物车中展示：缩略描述/图片 + 报价快照
- [x] 定制行进入与标准品相同的 checkout 流程
- [x] 报价快照在下单时锁定，不随后续 AI 调用变化

## UX

- **相关页面 / 流程**: 定制结果 → 购物车 → 结账
- **线框 / 布局**:

```
购物车中的定制行:
+------+
| 🎨  | [定制] 蝴蝶珍珠胸针   ¥380
|      | 「蝴蝶造型...」     数量: 1
+------+                     小计: ¥380
```

- **令牌 / 组件**: CartItem（定制品变体）, PriceSnapshot

## TL — 技术

- **ADR 引用**: ADR-2（OrderLine 统一接口）
- **实现备注**:
  - CartItem 结构扩展: `{ type: 'custom', customRequestId, description, imageUrl, quoteAmount }`
  - checkout 逻辑统一处理 standard / custom 两种行类型
  - 订单创建时将 quoteAmount 写入 OrderLine 快照字段
- **不纳入**: 定制行数量修改（MVP 固定为 1）；定制行退换

## QA

- **TCs**: TC-2.3
- **边界用例**: 重复加入同一定制项；定制行 + 标准品混合购物车总价计算；刷新页面后定制行是否保留
- **质量门禁**: 下单后 OrderLine 中 quoteAmount 与 AI 返回值一致

- [x] 验收标准全部满足
- [ ] TC-2.3 执行通过
