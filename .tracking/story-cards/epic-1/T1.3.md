# T1.3: 结账流程（Mock 支付）

## Summary

实现结账页面：展示订单摘要、收集收货信息、Mock 支付后跳转订单确认页。

## PM

- **里程碑**: M2
- **依赖**: T1.2（购物车提供结账入口）
- **风险 / 备注**: checkout 逻辑需为后续定制行复用预留空间。

## BA — 用户故事

**US-1.3** 作为 **用户**，我希望完成结账流程并看到订单确认，以便确认购买成功。

### 验收标准

- [x] 结账页展示订单摘要（商品 + 价格）
- [x] 填写收货信息（姓名、地址、手机号，前端校验即可）
- [x] 点击「提交订单」后 Mock 支付成功，跳转订单确认页
- [x] 支付为 Mock 实现，不接真实网关
