# T1.2: 购物车功能

## Summary

实现购物车：加购反馈、查看列表、修改数量、删除、计算总价，并提供「去结账」入口。

## PM

- **里程碑**: M2
- **依赖**: T1.1（商详页提供加购入口）
- **风险 / 备注**: 状态管理方案需兼容后续定制行扩展（T2.3）。

## BA — 用户故事

**US-1.2** 作为 **用户**，我希望将商品加入购物车并管理数量，以便一次性结账。

### 验收标准

- [x] 加入购物车后有视觉反馈（数量角标/toast）
- [x] 购物车页可查看已添加商品、修改数量、删除
- [x] 显示小计与总价
- [x] 有「去结账」按钮

## UX

- **相关页面 / 流程**: 购物车页
- **线框 / 布局**: 参见 design-specs.md — 购物车页线框
- **令牌 / 组件**: CartItem（标准品变体）, 数量选择器

## TL — 技术

- **ADR 引用**: ADR-2（OrderLine 统一接口）
- **实现备注**: 前端状态管理（Context / Zustand / Redux 视技术栈而定）；CartItem 数据结构预留 `type: 'standard' | 'custom'` 字段
- **不纳入**: 服务端持久化购物车（MVP 可用 localStorage / 前端状态）

## QA

- **TCs**: TC-1.2
- **边界用例**: 加入同一商品多次应合并数量；数量为 0 时自动移除或禁止；购物车为空时禁用结账
- **质量门禁**: 总价计算正确

## Definition of done

- [x] 验收标准全部满足
- [x] TC-1.2 执行通过
- [x] dashboard / tech-tasks 状态同步更新
