# T3.1: 订单创建与展示

## Summary

实现订单模型与确认页，统一展示标准品和定制品行项目，订单带有基础状态（已提交）。

## PM

- **里程碑**: M4
- **依赖**: T1.3（结账流程）, T2.3（定制行接入结账）
- **风险 / 备注**: 此任务完成即 MVP 全流程可演示。

## BA — 用户故事

**US-3.1** 作为 **用户**，我提交订单后希望看到订单详情，区分标准品和定制品。

### 验收标准

- [x] 订单确认页展示所有行项目
- [x] 标准品行显示：商品名 + 数量 + 单价
- [x] 定制行显示：定制描述摘要/图片缩略图 + 报价金额 + 「定制」标签
- [x] 订单有基础状态：已提交
- [x] 订单总价 = 标准品小计 + 定制品报价之和

## UX

- **相关页面 / 流程**: 订单确认页
- **线框 / 布局**: 参见 design-specs.md — 订单确认页线框

```
+------------------------------------------+
|  ✅ 订单已提交!                           |
|  订单号: #20260331-001                   |
+------------------------------------------+
|  商品A x1             ¥xx   [标准品]     |
|  蝴蝶珍珠胸针 x1      ¥380  [定制]      |
|  ─────────────────────                   |
|  合计: ¥ xxx                             |
|  状态: 已提交                             |
+------------------------------------------+
|  收货信息: 张三 / 138xxxx / 上海市...     |
+------------------------------------------+
|  [ 返回首页 ]                             |
+------------------------------------------+
```

- **令牌 / 组件**: OrderConfirmation, CartItem（只读模式）

## TL — 技术

- **ADR 引用**: ADR-2
- **实现备注**:
  - Order 模型: id, status('submitted'), shippingInfo, totalAmount, createdAt
  - OrderLine 模型: orderId, type('standard'|'custom'), productId?, customRequestId?, quantity, unitPrice, snapshotData
  - API: `POST /api/orders` (创建), `GET /api/orders/:id` (查询)
- **不纳入**: 订单列表页、状态流转（已发货/已完成）、订单通知

## QA

- **TCs**: TC-3.1
- **边界用例**: 纯标准品订单；纯定制品订单；混合订单；空订单（应被 checkout 拦截）
- **质量门禁**: 总价 = 各行小计之和；定制行报价与 AI 返回快照一致

## Definition of done

- [x] 验收标准全部满足
- [ ] TC-3.1 执行通过
- [x] dashboard / tech-tasks 状态同步更新
