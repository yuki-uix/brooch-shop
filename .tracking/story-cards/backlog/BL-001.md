# BL-001: AI 生成定制胸针效果图（设计预览图）

## Summary

在用户完成可行性/报价评估后（或作为可选步骤），由多模态/图像生成能力产出 **定稿向的效果预览图**（非必须等比例工程图），用于增强决策与沟通；与当前 MVP 仅返回「文字 + 数字报价」区分。

## PM

- **里程碑**: M4+（MVP 后迭代）
- **依赖建议**: T2.2（评估结构化结果稳定）、T2.3（可选，与购物车快照解耦也可先做纯展示）
- **风险 / 备注**: 生成图版权与平台政策、推理成本、与实物打样落差需免责声明；不宜替代正式设计稿。

## BA — 用户故事（草案）

作为 **用户**，希望在看到报价结论的同时（或之后），能看到 **接近成品风格的效果图**，以便判断是否继续下单或调整需求。

### 验收标准（草案）

- [ ] 明确入口：从定制结果页触发「生成效果图」或流程内可选步骤
- [ ] 输出可展示的图像（URL 或 base64 落盘），并与 `customRequest` 关联存储
- [ ] 前台展示加载态与失败兜底；重复生成策略（限次/覆盖）可配置
- [ ] 免责声明：效果图供参考，以实物打样为准

## TL — 技术（草案）

- 候选：`@ai-sdk/google` / OpenAI Images / 其他图像生成 API；与现有 `evaluate` 分流，新路由如 `POST /api/custom-requests/:id/render`。
- 输入：现有 description + imageUrl + 评估结果摘要（可选），Prompt 固定模板 + 合规过滤。
- **不纳入（首版）**: 矢量工程图、多视角一致性强约束、用户实时编辑图层。

## UX（草案）

- 结果页增加「预览效果图」区块或次级页；与 design-specs 后续增补线框对齐。
