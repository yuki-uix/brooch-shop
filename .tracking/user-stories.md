# 用户故事

## MVP 范围

### 纳入范围

- 胸针为唯一品类，支持按材质（珍珠/贝母/纯银/锆石等）分类与筛选
- 标准品完整交易链路：浏览 → 商详 → 加购 → 结账（支付 Mock）
- 定制入口：用户上传图片或文字描述 → AI 判断可行性并报价 → 进入统一 checkout
- 订单展示：区分标准品行与定制行，含报价快照

### 不纳入范围（Epic 4 明确排除）

- 通过 Agent 完成 Checkout（用户仍手动进入 Checkout 页）
- 定制询价对话（属于 Epic 2，维持原有独立流程）
- 订单查询 / 修改（属于未来 Epic）
- 跨 session 对话历史持久化（刷新即清空，Phase 3 再做）
- 基于购买历史的个性化推荐（无登录体系，MVP 不做）
- 库存实时查询（种子数据不含库存字段，不引入）

### 不纳入范围（Phase 2+）

- 多轮对话改稿、设计师人工审核工单
- 报价与真实成本/工时深度挂钩的定价引擎
- 多租户完整隔离、复杂权限体系
- 多语言、SEO、营销自动化
- 用户注册/登录（MVP 可用访客身份或简化 session）
- 退换货、售后流程

## 角色 / 参与者

- **访客/用户**: 浏览、筛选、购买标准品胸针或提交定制需求
- **AI 服务**: 接收图片/文字，返回可行性判断与报价
- **系统**: 管理商品、材质、订单数据

## Epics 与故事

### Epic 0 — 素材与分类

#### US-0.1 材质数据与商品属性

作为 **系统管理者**，我需要一套材质枚举与商品数据模型，以便前台可以按材质对胸针进行分类展示。

**验收标准**

- [ ] 材质表包含至少 4 种：珍珠、贝母、纯银、锆石
- [ ] 商品与材质之间存在关联（单选或多选）
- [ ] 可通过种子数据生成至少 8 件示例商品

#### US-0.2 按材质筛选商品列表

作为 **用户**，我希望在商品列表页按材质筛选胸针，以便快速找到我喜欢的材质类型。

**验收标准**

- [ ] 列表页显示材质筛选入口（标签/下拉）
- [ ] 选择材质后，列表仅展示对应商品
- [ ] 可清除筛选恢复全部商品
- [ ] 每件商品卡片上显示材质标签

### Epic 1 — 标准品交易

#### US-1.1 商品详情页

作为 **用户**，我希望点击商品进入详情页，查看大图、价格、材质、描述等信息，以便做购买决策。

**验收标准**

- [ ] 展示商品名称、价格、材质、描述、主图
- [ ] 有「加入购物车」按钮
- [ ] 材质信息与列表页一致

#### US-1.2 购物车

作为 **用户**，我希望将商品加入购物车并管理数量，以便一次性结账。

**验收标准**

- [ ] 加入购物车后有视觉反馈（数量角标/toast）
- [ ] 购物车页可查看已添加商品、修改数量、删除
- [ ] 显示小计与总价
- [ ] 有「去结账」按钮

#### US-1.3 结账（Mock 支付）

作为 **用户**，我希望完成结账流程并看到订单确认，以便确认购买成功。

**验收标准**

- [ ] 结账页展示订单摘要（商品 + 价格）
- [ ] 填写收货信息（姓名、地址、手机号，前端校验即可）
- [ ] 点击「提交订单」后 Mock 支付成功，跳转订单确认页
- [ ] 支付为 Mock 实现，不接真实网关

### Epic 2 — 定制询价

#### US-2.1 定制入口与输入界面

作为 **用户**，我希望通过「定制」入口上传图片或输入文字描述，以便发起定制胸针请求。

**验收标准**

- [ ] 首页/导航有明确的「定制」入口
- [ ] 定制页支持上传图片（JPG/PNG，≤5MB）
- [ ] 定制页支持文字描述输入（≤500字）
- [ ] 图片和文字可单独使用或组合提交
- [ ] 空提交时给出提示

#### US-2.2 AI 可行性判断与报价

作为 **用户**，我提交定制需求后，希望看到 AI 给出的可行性结论和报价，以便决定是否继续。

**验收标准**

- [ ] 提交后展示 loading 状态
- [ ] 返回可行性判断：「可制作」/「需确认」/「不建议制作」三级
- [ ] 返回简短理由说明
- [ ] 返回报价金额（数字）或价格区间
- [ ] 页面包含免责声明（如「此为参考报价，最终价格以确认为准」）
- [ ] API 超时/失败时展示友好错误提示

#### US-2.3 定制项进入结账

作为 **用户**，我接受 AI 报价后，希望将定制项加入购物车或直接结账，以便完成定制下单。

**验收标准**

- [ ] 「可制作」结果旁有「加入购物车」/「立即结账」按钮
- [ ] 定制行在购物车中展示：缩略描述/图片 + 报价快照
- [ ] 定制行进入与标准品相同的 checkout 流程
- [ ] 报价快照在下单时锁定，不随后续 AI 调用变化

---

### Epic 3 — 订单

#### US-3.1 订单创建与展示

作为 **用户**，我提交订单后希望看到订单详情，区分标准品和定制品。

**验收标准**

- [ ] 订单确认页展示所有行项目
- [ ] 标准品行显示：商品名 + 数量 + 单价
- [ ] 定制行显示：定制描述摘要/图片缩略图 + 报价金额 + 「定制」标签
- [ ] 订单有基础状态：已提交
- [ ] 订单总价 = 标准品小计 + 定制品报价之和

---

### Epic 4 — 购物助手 Agent

#### US-4.1 打开购物助手

作为 **用户**，我希望在购物页随时打开 AI 购物助手侧栏，以便在不离开商品列表的情况下获得帮助。

**验收标准**

- [ ] 导航栏或右下角有「AI 助手」触发按钮
- [ ] 点击后从右侧滑入 Chat Panel，不遮挡主内容区域
- [ ] 可随时关闭，关闭后对话历史本次 session 内保留
- [ ] 移动端 Panel 全屏展示

#### US-4.2 自然语言搜索商品

作为 **用户**，我可以用自然语言描述需求，助手返回匹配的商品列表。

**验收标准**

- [ ] 输入「适合婚礼 + 珍珠 + 300以内」，助手调用 `searchProducts` 工具并以卡片行展示结果
- [ ] 每张卡片含：图片、名称、材质标签、价格
- [ ] 搜索结果为空时，助手给出友好回应并提供替代建议
- [ ] 工具调用过程中显示「正在搜索商品…」中间状态

#### US-4.3 追问与多轮对话

作为 **用户**，我追问「换成银材质的」，助手能保持上下文重新搜索，不需要我重新说价格范围。

**验收标准**

- [ ] `useChat` messages 数组在本次 session 全程保持
- [ ] 助手可正确引用前轮意图（材质变更但价格范围延续）
- [ ] 用户说「介绍一下第二个」，助手调用 `getProductDetails` 并展示完整信息

#### US-4.4 Agent 直接加购

作为 **用户**，我说「帮我把第一个加入购物车」，助手完成操作并给出确认，购物车 badge 同步变化。

**验收标准**

- [ ] 助手调用 `addToCart` 客户端工具，Zustand store 立即更新
- [ ] 导航栏 cart badge 动画更新
- [ ] 聊天内出现「已将 [商品名] 加入购物车 ✓」确认消息
- [ ] 加购失败（商品 ID 无效等）时，助手报告错误原因
